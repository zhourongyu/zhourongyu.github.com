Scala学习-模式匹配

發表於 2014-06-11   |   分類於 tech   |  

模式匹配

模式匹配是scala的一大特性，有点像增加版的switch-case。
在Scala代码中无处不在：用于合并条件执行、解构(destructuring) 、在构造中造型。 

def getSex(name:Any){
    name match {
        //匹配字符串
          case "rony" => println(“male”)
          case "lina" => println(“female”)
          //匹配元组
          case ("rony","lina") => println(name)
          //匹配列表
          case List("rony","lina") => println(name)
          //匹配自定义类
          case User(u) => println(u.name)
          //匹配通配符
          case _ =>
    }
  }

《Effective Scala》中建议匹配通配符时改为 case other => 可读性更高

Options

写多了java得同学都见多了NullPointerException，一不小心就会出现这种低级错误，Scala中有一种Option类型，是一个表示有可能包含值的容器。空(None)或满(Some(value))二选一。

《Effective Scala》中提出Option提供了使用null的另一种安全选择，应该尽可能的替代null。它是一个集合(最多只有一个元素)并用集合操所修饰，尽量用Option。
但它也提示了不要过度使用Option： 如果有一个明确的缺省值——一个Null对象——直接用Null而不必用Option

def getUser(user_id: Long):User = {
    //userDao.getUser()返回Option[User]类型
    val user = userDao.getUser(user_id)
    user match {
        case Some(u) => u
        case None => //不存在用户
    }
}

有人说这不和if判断null一样么，根本没有什么优势，个人认为这只是更像一种警示，当习惯后，看到方法的返回值是Option时，心里应该明白这个可能是要做空处理的。
Option里面还有一些的好用方法，例如getOrElse()，如果没有取到则给一个默认值，

def getUserName(user_id: Long):String = {
    //userDao.getUserName()返回Option[String]类型
    val user = userDao.getUserName(user_id)
    user.getOrElse("rony")
}

2014大连徒步大会

發表於 2014-05-17   |   分類於 photo   |  

大连国际徒步大会，每年五月份在大连举办，今年已经是十二届了，自从毕业后再也没参加过了。
在这个全民健身的时代，徒步大会在很多地方都有举办，但大连这场独具特色，很大原因是地理位置，具有天然的优势，整个徒步路线沿经大连最美道路之滨海路，途经星海广场、贝壳博物馆（已经改成酒店了，可惜）、森林动物园、付家庄海滨公园、老虎滩、海之韵。每个景点都是大连的特色。

叫上久违的哥们一起造起来，预计是30km的路程，正常的徒步速度大概4-5km/h，这种游玩场合比较慢，早上七点半火车到，到宾馆放行李集合吃饭出发，已然已经十点左右了~ 正常的30km出发时间为早上六点…….

弱智青年欢乐多，工作后难得有如此放松的时候，像个2B…
如果日后我们都成有钱人了，不知道还没有时间再一起疯了~~~

一路玩耍到六点才到20km处，走完30估计要走夜路了~ 不得不放弃了30km得计划，大家也非常的累，大概年事已高……
返程后接着就是大餐，夜聊。
回大连有种回家的感觉，每一处都是那么的熟悉，似乎处处都有回忆。
人生就是这样，明年再来。

五月飞雪长白山

發表於 2014-05-05   |   分類於 photo   |  

长白山大约有半年时间处于冰雪期，这次五一来这里也算赶上了末班车。据当地居民了解，前几天刚好下了大雪，虽然劳累一天奔波在路上，但因为第一天天气特别好，并且天气预报明日有雨，随即放弃吃饭休息，直奔景区（14：30停止进山）。

长白山景区分西坡/南坡/北坡三大块，北坡最早开发，设施也很齐全。第一天我们去的是西坡，这边人比较少，为了省时间直奔天池。山顶白茫茫的雪，天池也处于大冰块状态，上面浮着大片雪花。

西坡景点比较少，但可以看到天池全景。第二日我们再去往北坡，遗憾的是从五一开始门票只限当日进山了，只能再多花125门票！
而北坡最坑爹的是不只需要买门票和摆渡车，上天池需要另付一趟越野车费。以至于我们放弃再登顶，去长白山瀑布/地下森林等景点游玩了

总体来说天池确实很有特色，至于其他的什么大峡谷/森林都不太值得一去。
这次由于是自驾游，悠闲随意，让我们惊喜的是整个旅途非常顺畅，甚至高速上没见到几辆车，完全没有国内假期旅游的景象，也许我们真选对了地方，如果每次都这样那就太好了。

因为自驾的原因，路途中遇见美丽的地方都会下来游玩一下。印象深刻的除了沈吉/抚长高速上那些个性特色的服务站，还有整个途中大片的白桦林，在景区附近的省道/国道/小道路边全是。加上经常一俩车都看不到，感觉像是来到了国外，令人心旷神怡。

武汉新映像

發表於 2014-04-10   |   分類於 photo   |  

清明我从一座雾霾城市「北京」来到了另一座雾霾城市「武汉」，真是一担两桶，半斤八两，霾的厉害。
几年前我第一次去武汉，给我的映像非常差，可用脏乱差来形容。
窄小的马路和低矮的楼房。
装修还不如我家县城的统一两元的豪华空调公交，毫无秩序的乘车人。

凭借这样的映像来到武汉也是意外，本打算去长白山拍“水怪”去，种种原因却来了武汉。既来之则安之，不过这次的映像好很多，起码两元坐上了干净舒适的公交~不变的依旧是那乘车的人，似乎从来没见过排队的景象。
武汉什么好吃，周黑鸭？No，是烤虾… 武汉本地一朋友带我们吃了「水水烤虾」，确实很好吃。类似北京的麻小（麻辣小龙虾），不过比北京的好吃实惠的多。烧烤味道都很不错，火锅味道也还行，比较出名的「三国英雄」。三天里我们共吃了三顿烧烤，一顿火锅。

因为雾霾天，挺影响拍片的。以至于在东湖里竟然一张风景片没出，每晚坚持去了趟长江大桥，而且还去晚了~~ 不过还是出了张能看的

上面这张是拍摄的经典位置，位于桥下岸边。下面这张站于露台上。 这次没有拍上二桥的景色，长江二桥也很漂亮，虽然没有大桥这样宏伟壮观。

拍摄注意事项: 桥上观景灯关闭时间为九点半…
旅游假日竟然这么早关灯，幸好去的不是太晚，不然毛都没有了！

在武汉游玩景点相当集中，都集中在长江大桥两侧，西边汉口区的江汉路/龟北路/江滩，东边的户部巷/黄鹤楼/昙华林… 只是东湖稍远，磨山景区里有滑道，比索道有意思多了，可以想象为一个人坐在一个盆里冲下山底…乐趣十足。

Guava Cache

發表於 2014-04-09   |   分類於 tech   |  

Google Guava Collections（简称 Guava）是Google推出的Java类库，针对JDK做了很多增强和扩展，例如Map的K-V转换，ConcurrentHashMap的构造等。可以有效使你的代码量更简洁高效。

Guava Cache是一个全内存的本地缓存实现，提供了线程安全的实现机制。Guava Cache本身是一个Map,实现的机制类似ConcurrentHashMap。

官方注解中提出在以下需求中可能会使用到它：

  • automatic loading of entries into the cache
  • least-recently-used eviction when a maximum size is exceeded
  • time-based expiration of entries, measured since last access or last write
  • keys automatically wrapped in WeakReference
  • values automatically wrapped in WeakReference or SoftReference soft
  • notification of evicted (or otherwise removed) entries
  • accumulation of cache access statistics

自动装载，基于时间过期，最少使用排除等，恰好符合最近项目中需要解决一个性能的小问题需求。

Guava提供两种类型缓存：LoadingCache&Cache。LoadingCache可以当entry不存在时加载entry，Cache不会自动加载entry。

private LoadingCache<String, Map> topic_cache_map = CacheBuilder.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(5, java.util.concurrent.TimeUnit.MINUTES)
        .build(
                new CacheLoader<String, Map>() {
                    public Map load(String key) throws Exception {
                        if("-1".equals(key)) {
                            return getLastTopic();
                        }else{
                            return getTopic(key);
                        }
                    }
                });

设置最大数量1000，五分钟过期时间。 直观的效果体现请参照下图
