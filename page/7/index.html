2014 Mac装机手册

Veröffentlicht am 2014-01-14   |   in tech   |  

2014年最新的Mac新机配置单，仅供参考

基础组件：

Homebrew

Homebrew是mac下最好用的包管理器，安装方式：

ruby -e "$(curl -fsSL https://raw.github.com/Homebrew/homebrew/go/install)"

eg: brew install wget

oh-my-zsh

zsh是mac下bash的替代品，或者说是bash的增强版，有不少强大的插件。 oh-my-zsh是zsh常用的配置包。

iTerm2

iTerm是Mac自带终端的增强版

开发工具

Command Line Tools：基本开发命令工具集合
Dash：语法库文件本地查询与生成工具
Github for Mac：Github官方客户端
IntelliJ IDEA：对mac支持最好的JAVA开发工具

通用软件

云端软件：
Evernote：强大个人云笔记，横跨各个平台
Dropbox：文件云端同步利器

输入法：
百度输入法：百度输入法是最近换过来的，经过试用和QQ输入法旗鼓相当
qq输入法：稳定，更新教快

写作与编辑：
Sublime Text 3：编辑器神作
Mou：简洁，实时预览的Markdown写作神器

系统增强：
Alfred 2：必装神器之一，配合work flow可出现神龙
Keka：免费压缩/解压软件
Thunder：比Windows下简洁太多，好太多
MPlayerX：影音播放器

社交：
QQ：社交老大

Aperture 3：苹果自家强大相片管理+调整软件，iPhoto增强版。

Apple特别购物日

Veröffentlicht am 2014-01-12   |   in life   |  


今天是苹果优惠购物日，每年苹果会在世界各地举办新年优惠活动，不同国家方式不同。比如日本的福袋就挺有意思…大陆的活动历来感觉一般。去年记得ipad什么的减价大约是300左右。今年果然也差不多~让苹果打个折太不容易了…

在今天下手了一台ME294，国行官网订购，最大的好处是招行/工行的分期0手续费，相当的给力。最大支持12期0手续费，如果把这些钱放到余额宝里，按现在的收益率6%来算。 6%*18000 = 1000元左右，也就是说，在优惠700元的前提下，还能省下好几百。

重量2.02kg，比普通的13寸MBP还要稍轻一些，当然还是比Air重一斤多。总体还是非常能接受的。

关于Retina屏幕的效果，记得之前在Apple Store里看，并没有觉得很惊艳。
到手后特意和自己的MBP做了仔细对比，在相同设置下访问同一网站/网页，咋看起来差不太多，但仔细一看会感觉RMBP非常锐，越看越想…可以就像看单反和卡片拍出相片~感觉一切都是那么的漂亮。

经过几天的使用，我确实再也不想看MBP的屏幕了…连自己的Air也丢到一边去了！

这是一台买了绝对不会后悔的神器，2013令人最具幸福感产品之一！

由Wordpress迁移至Typecho

Veröffentlicht am 2013-12-10   |   in tech   |  

Typecho

关注Typecho是在去年，在用了这么久得Wordpress后感觉累了，写文章得不方便+巨多得插件管理等，确实略显庞大。
Typecho是一个轻量级得php程序，在我关注得时候就想试试，但一看已经两年没有更新了～ 当时Sunny说快要有新版本了，所以一直关注中…
前不久0.9的测试版发布，憋到现在正式版终于出山了，立即切换了过来。
再关注得期间也想尝试Jekyll/Octopress ，但因为是Ruby得东西，实在有点不适应，所以放弃。

Markdown

这几个项目有共同得特色，就是支持Markdown语法，极大得简化了写作得「成本」，Wordpress一直让我诟病的就是写文章时候得麻烦，网页基本无法写东西（排版太难）…在本地使用Live Writer是公认最为合适得写作方案，但对于图片等文件得支持又要麻烦些。
如果博客连基本得写作都不能让人舒心，就算做得功能再全，效果再好那也不能得到写作者得青睐。只可能会更吸引一些好奇得读者。

而Markdown得出现就是为了不再忧虑排版，专注写作，学习成本也极低。

写作

目前真正最省心高效得方法应该是Github+Jekyll.连服务器都不用管，只用专心写作，重要得是完全免费……
大部分人做事都是由繁入简的，刚开始玩博客你会想尽办法尝试新得东西，进入无限得折腾，当时间长了以后部分人开始放弃写博客了，因为没有折腾得动力/兴趣了，他们喜欢得是这个过程，而不是写作。而另外一些对写作有一定得热爱的人开始由繁入简，换简单得模板，用简介得写作方案。

下一步要进行得是内容得转移，这是一个大工程，估计又要折腾不少时间。

Nginx反向代理获取真实IP

Veröffentlicht am 2013-11-27   |   in tech   |  

今天面试官的一个问题让我有点疑惑：如何配置 Nginx 能让程序在后台获取真实的用户IP，当时我没理解这个问题意思，获取请求IP不是很简单的事情么？例如：request.getRemoteAddr()

回来查询了资料. 此问题是指使用了 Nginx 反向代理获取不到真实用户的 IP ，当使用了反向代理后 getRemoteAddr() 返回的是反向服务器的 IP。

什么是反向代理，曾经写了一个聊天小应用并部署再服务器上，端口为8888，本地访问也就是127.0.0.1:8888

我想让 t.imim.me 访问到我的这个应用上（应用已撤除），配置如下

proxy_pass http://chat

这就是反向代理，将来自这个域名的请求转发到了一个名为 chat 的 upstream，也是真实的应用地址（127.0.0.1:8888）

再看这两行配置:

proxy_set_header X-real-ip $remote_addr
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

配置 X-real-ip 就可以在 web 服务器端通过 request.getAttribute(“X-real-ip”) 获得用户的真实 ip。

X-Forwarded-For 是一个 proxy 转发记录,每次经过转发都会有记录，以 client,proxy1,proxy2 格式保存。如果没有2配置的话，后端通过 request.getAttribute(“X-Forwarded-For”) 获取值为Null。

当一个应用经过 N(N>1) 次 Nginx 转发时，X-real-ip 中保存的是 N-1 的服务器地址，也就是说当第二次代理的时候 X-real-ip 的值为第一台 Nginx 服务器 IP，X-Forwarded-For 的值则变为”用户真实 IP , 第一台 Nginx 服务器 IP”
