Comparator vs Comparable

發表於 2014-12-03   |   分類於 tech   |  

Java中对对象的集合或数组进行排序时通常有两种方式：Comparator && Comparable。

Comparable要求使用对象实现其compareTo方法来实现自定义的排序方式。
但在使用的场景中，往往在设计时并没有考虑到比较问题从而没有实现Comparable，这时候再去改造对象本身或许不太合适。

Comparator更加的灵活，采用的是策略模式，在不改变对象本身的情况下来改变对象的行为
举例说明：

class User {
    //年龄
    private Integer age;
    //身高
    private Integer height;
    //姓名
    private String name;

    public Integer getAge() {
        return age;
    }

    public void setAge(Integer age) {
        this.age = age;
    }

    public Integer getHeight() {
        return height;
    }

    public void setHeight(Integer height) {
        this.height = height;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

现在需要给所有的User对象按年龄从小到大排序，如果年龄相同则按身高由高到低来排序。
以下是Comparable方式，通过改造User对象来实现自定义compileTo方法：

class User implements Comparable<User>{
    //年龄
    private Integer age;
    //身高
    private Integer height;
    //姓名
    private String name;

    public Integer getAge() {
        return age;
    }

    public void setAge(Integer age) {
        this.age = age;
    }

    public Integer getHeight() {
        return height;
    }

    public void setHeight(Integer height) {
        this.height = height;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public int compareTo(User u) {
        int flag = this.age.compareTo(u.getAge());
        if (flag == 0 ) {
            return (this.height.compareTo(u.getHeight())) * -1;
        } else {
            return flag;
        }
    }
}

以下是Comparator的实现方法及测试类：

public class Test {
    public static void main(String[] args) {

        User user1 = new User();
        user1.setAge(20);
        user1.setHeight(180);
        user1.setName("张三");
        User user2 = new User();
        user2.setAge(20);
        user2.setHeight(176);
        user2.setName("李四");
        User user3 = new User();
        user3.setAge(25);
        user3.setHeight(155);
        user3.setName("王二");
        User user4 = new User();
        user4.setAge(23);
        user4.setHeight(155);
        user4.setName("麻子");
        List<User> userList = new ArrayList<User>();
        userList.add(user1);
        userList.add(user2);
        userList.add(user3);
        userList.add(user4);

        //使用Comparator实现排序
        Collections.sort(userList, new Comparator<User>() {
            @Override
            public int compare(User u1, User u2) {
                int flag = u1.getAge().compareTo(u2.getAge());
                if (flag == 0) {
                    //身高降序
                    return (u1.getHeight().compareTo(u2.getHeight())) * -1;
                } else {
                    return flag;
                }
            }

        });

        //如果使用Comparable方式，可直接使用以下方式排序即可
        Collections.sort(userList);

        for (User u : userList) {
            System.out.println("姓名："+u.getName()+"，年龄："+u.getAge()+"，身高："+u.getHeight());
        }

    }

}

结果:

姓名：张三，年龄：20，身高：180
姓名：李四，年龄：20，身高：176
姓名：麻子，年龄：23，身高：155
姓名：王二，年龄：25，身高：155

达到的效果一致，但是，这并没有结束。
在Java规范里，倡导保证equals和compareTo的行为一致，这样可以避免出现一些奇奇怪怪的问题。大致意思就是如果user1.compareTo(user2) == 0 的话，user1.equals(user2) == 1
既然重载了equals，那意味着hashcode也得重载了~ 你懂的

从清迈到Pai

發表於 2014-10-09   |   分類於 photo   |  

大部分去了清迈的年轻人都会去Pai，这是一个充满着小清新的小镇，城区也非常之小。
总距离130km左右，中途包含30km的107国道，近100km的1095国道，注意：这100km基本上全是山路，一千多个弯道和众多的180°急弯。

查过攻略的都知道，去Pai一般是乘坐mini bus，150-200B/人。价钱不贵，重点是因为山路太崎岖，坐车非常难受，亲眼目睹不少从车上下来的人直接在那呕吐…

因为我是骑摩托去的，所以免去了晕车的痛苦，途中也相当的刺激和兴奋，也是第一次骑这么远而险的路。背着满满的行李还带个妹纸，所幸车技还不错，安全抵达，抵达后没几个小时发现后轮胎被扎破，很长一颗钉子，也算运气好了，如果是山路中这样估计就要磨死人了。

和自己的小摩托合影，自行前往的好处除了舒服，另外就是自由，一路上绝美的风景可以细心玩耍。

总体时间上来说也并不比bus慢多少，正常的mini bus时间约在三个半小时，我们总时长四个多小时，包含中途休息拍照片时间~
强烈建议有「能力」的童鞋一定要骑行前往。

关于Pai的景点网上一查便知，我强烈推荐的是「云来观景台」，仅此一个，在Pai两天时间，我一共去了四次，当然很大原因是想拍几张大片，在上面可以看到整个Pai县，因为面朝东边，也是绝佳的日出观赏点，因为期间除了下雨就是起雾，四次登顶全部无功而返，这是此行最大的遗憾（谁让自己要在雨季前往呢~）

这是在清晨的雾气散过之后，终于拍到了几张照片。云来门票20铢，相当于人民币四元左右，还有免费续杯的茶水，据管理员说这是乌龙茶~ 不论是清晨或是傍晚，坐在这里品着茶水，看着大朵大朵的云彩流动，可以忘却一切烦恼。

管理员是一位中国老大爷，山地村人。山地村村民是国民党时期逃过去的同胞，直到今天很多人的普通话已经很生疏了，虽然发音很标准，但语句已然不太通顺~


在这个充满文艺气息的小镇非常容易出片…也难怪众多Chinese girl特别的喜欢这里。

在我房间阳台拍的外景，这样的环境国内不过K应该住不到吧~ 这里只需不到200~

Pai县的夜市很简单但东西也比较全，晚上夜市小街上做Message(马杀鸡)，顺便和服务员聊天，“泰国在两年前并不是很多人来，至《泰囧》之后，大批的中国人到来~”。 据这几天观察大约有80%以上的游客是中国人。

照片摄于「红海滩」，是盘锦少数几个具有可玩性的地方之一~ 面积非常大，前半段两侧都是大片大片的芦苇荡子，让我想起了白洋淀。

滩内是一大片湿地，滩内的红色也是一种碱蓬草，每年九/十月份的时候颜色最深，呈红紫色。也是最漂亮的时候。除了这些草，里面爬满了螃蟹，也是盘锦另一大特色「盘锦河蟹」。

河蟹虽小，五脏俱全~ 二两多算是比较大的蟹了。肉自然没有海蟹那么多，不过小的优势在于比较入味，还是非常好吃，价格大约在80~100/kg。感谢当地朋友的盛情款待，让我们吃了一顿免费的螃蟹~~~

jvm的时区问题

發表於 2014-09-02   |   分類於 tech   |  

最近发现在一台开发机上java程序启动后时间不对，比系统时间慢八小时，检查系统时间却发现是正确的…

有俩种解决方案，方案一:

//程序内设置TimeZone
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.TimeZone;
class Test{
    public static void main(String[] args){
        SimpleDateFormat df = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        TimeZone tz = TimeZone.getTimeZone("Asia/Shanghai");
        df.setTimeZone(tz);
        System.out.println(df.format(new Date()));
        System.out.println(new Date().toString());
    }
}

result：

2014-09-02 23:03:25
Tue Sep 02 15:03:25 UTC 2014

方案二：

在程序的启动脚本里加上时区参数
-Duser.timezone=Asia/Shanghai

打印结果可以看出是时区不正确，jvm获取到的时区为UTC，而正确的时间应为CST（北京时间），方案二显然更加灵活但也治标不治本。
接下来检查当前系统属性

//遍历当前系统属性
final Properties p = System.getProperties();
final Enumeration e = p.keys();
while (e.hasMoreElements()) {
    final String prt = (String) e.nextElement();
    final String prtvalue = System.getProperty(prt);
    System.out.println(prt + ":" + prtvalue);
}

终极方案：修改系统的时区为「Asia/Shanghai」。

cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
