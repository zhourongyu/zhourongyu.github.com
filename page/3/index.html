从清迈到Pai

發表於 2014-10-09   |   分類於 photo   |  

大部分去了清迈的年轻人都会去Pai，这是一个充满着小清新的小镇，城区也非常之小。
总距离130km左右，中途包含30km的107国道，近100km的1095国道，注意：这100km基本上全是山路，一千多个弯道和众多的180°急弯。

查过攻略的都知道，去Pai一般是乘坐mini bus，150-200B/人。价钱不贵，重点是因为山路太崎岖，坐车非常难受，亲眼目睹不少从车上下来的人直接在那呕吐…

因为我是骑摩托去的，所以免去了晕车的痛苦，途中也相当的刺激和兴奋，也是第一次骑这么远而险的路。背着满满的行李还带个妹纸，所幸车技还不错，安全抵达，抵达后没几个小时发现后轮胎被扎破，很长一颗钉子，也算运气好了，如果是山路中这样估计就要磨死人了。

和自己的小摩托合影，自行前往的好处除了舒服，另外就是自由，一路上绝美的风景可以细心玩耍。

总体时间上来说也并不比bus慢多少，正常的mini bus时间约在三个半小时，我们总时长四个多小时，包含中途休息拍照片时间~
强烈建议有「能力」的童鞋一定要骑行前往。

关于Pai的景点网上一查便知，我强烈推荐的是「云来观景台」，仅此一个，在Pai两天时间，我一共去了四次，当然很大原因是想拍几张大片，在上面可以看到整个Pai县，因为面朝东边，也是绝佳的日出观赏点，因为期间除了下雨就是起雾，四次登顶全部无功而返，这是此行最大的遗憾（谁让自己要在雨季前往呢~）

这是在清晨的雾气散过之后，终于拍到了几张照片。云来门票20铢，相当于人民币四元左右，还有免费续杯的茶水，据管理员说这是乌龙茶~ 不论是清晨或是傍晚，坐在这里品着茶水，看着大朵大朵的云彩流动，可以忘却一切烦恼。

管理员是一位中国老大爷，山地村人。山地村村民是国民党时期逃过去的同胞，直到今天很多人的普通话已经很生疏了，虽然发音很标准，但语句已然不太通顺~


在这个充满文艺气息的小镇非常容易出片…也难怪众多Chinese girl特别的喜欢这里。

在我房间阳台拍的外景，这样的环境国内不过K应该住不到吧~ 这里只需不到200~

Pai县的夜市很简单但东西也比较全，晚上夜市小街上做Message(马杀鸡)，顺便和服务员聊天，“泰国在两年前并不是很多人来，至《泰囧》之后，大批的中国人到来~”。 据这几天观察大约有80%以上的游客是中国人。

“Perfect，这俩真可爱”
说完赶紧跑

照片摄于「红海滩」，是盘锦少数几个具有可玩性的地方之一~ 面积非常大，前半段两侧都是大片大片的芦苇荡子，让我想起了白洋淀。

穿过近10km路，随后进入眼帘的是大片的红色，满屏幕的红色~。

虽然很美，但并不太好下手拍摄，没有高点，网上图片大部分都是航拍的~要是有座山丘就好了。旁边有座大船，其实是个饭馆~

滩内是一大片湿地，滩内的红色也是一种碱蓬草，每年九/十月份的时候颜色最深，呈红紫色。也是最漂亮的时候。除了这些草，里面爬满了螃蟹，也是盘锦另一大特色「盘锦河蟹」。

河蟹虽小，五脏俱全~ 二两多算是比较大的蟹了。肉自然没有海蟹那么多，不过小的优势在于比较入味，还是非常好吃，价格大约在80~100/kg。感谢当地朋友的盛情款待，让我们吃了一顿免费的螃蟹~~~

jvm的时区问题

發表於 2014-09-02   |   分類於 tech   |  

最近发现在一台开发机上java程序启动后时间不对，比系统时间慢八小时，检查系统时间却发现是正确的…

有俩种解决方案，方案一:

//程序内设置TimeZone
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.TimeZone;
class Test{
    public static void main(String[] args){
        SimpleDateFormat df = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        TimeZone tz = TimeZone.getTimeZone("Asia/Shanghai");
        df.setTimeZone(tz);
        System.out.println(df.format(new Date()));
        System.out.println(new Date().toString());
    }
}

result：

2014-09-02 23:03:25
Tue Sep 02 15:03:25 UTC 2014

方案二：

在程序的启动脚本里加上时区参数
-Duser.timezone=Asia/Shanghai

打印结果可以看出是时区不正确，jvm获取到的时区为UTC，而正确的时间应为CST（北京时间），方案二显然更加灵活但也治标不治本。
接下来检查当前系统属性

//遍历当前系统属性
final Properties p = System.getProperties();
final Enumeration e = p.keys();
while (e.hasMoreElements()) {
    final String prt = (String) e.nextElement();
    final String prtvalue = System.getProperty(prt);
    System.out.println(prt + ":" + prtvalue);
}

本地使用的jdk环境是64-Bit jdk7U67，系统环境64Bit Ubuntu 12.04。
终极方案：修改系统的时区为「Asia/Shanghai」。

cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

Nginx的模块安装

發表於 2014-08-04   |   分類於 tech   |  

在给nginx增加反代的时候抛出如下错误：
nginx: [emerg] unknown directive "sub_filter"
检查后发现是没有安装http_sub_module 模块，此模板用于过滤/替换响应及一些内容
nginx模块的增加方式比较便捷，首先，nginx -V 查看当前nginx的编译参数

[root@something ~]# /usr/local/nginx/sbin/nginx -V
nginx version: nginx/1.6.0
built by gcc 4.8.2 20140120 (Red Hat 4.8.2-16) (GCC)
TLS SNI support enabled
configure arguments: --user=www --group=www --prefix=/usr/local/nginx --with-http_stub_status_module         
--with-http_ssl_module --with-http_gzip_static_module --with-ipv6

进到nginx安装源码目录，在老的编译参数中增加http_sub_module模块，并编译。
如果是第三方模块，则需要添加「—add-module=模块的源码路径」，这里以openresty的echo模板为样例（echo模块可以让nginx输出一些简单信息，方便调试）

[root@something nginx-1.6.0]# ./configure  --user=www --group=www --prefix=/usr/local/nginx --with-http_stub_status_module --with-http_sub_module --with-http_ssl_module --with-http_gzip_static_module --with-ipv6 --add-module=/home/z/ngx_openresty-1.7.2.1/build/echo-nginx-module-0.54
[root@something nginx-1.6.0]# make

只进行make，如果make install则所有文件被覆盖，导致conf丢失。
停止服务，备份老的nginx脚本，并用objs目下新的脚本替换掉老版本。

[root@something nginx-1.6.0]# cp /usr/local/nginx/sbin/nginx /usr/local/nginx/sbin/nginx.bak
[root@something nginx-1.6.0]# cp ./objs/nginx /usr/local/nginx/sbin/nginx

启动服务… 查看新的编译参数

[root@something nginx-1.6.0]# /usr/local/nginx/sbin/nginx -V
nginx version: nginx/1.6.0
built by gcc 4.8.2 20140120 (Red Hat 4.8.2-16) (GCC)
TLS SNI support enabled
configure arguments: --user=www --group=www --prefix=/usr/local/nginx --with-http_stub_status_module 
--with-http_sub_module --with-http_ssl_module --with-http_gzip_static_module 
--with-ipv6 --add-module=/home/z/ngx_openresty-1.7.2.1/build/echo-nginx-module-0.54

大功告成！
